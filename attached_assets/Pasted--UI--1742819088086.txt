以下のコードでは、店舗情報を入力・編集するための項目は概ねそろっていますが、まだ追加すると便利・わかりやすい情報やUI改善の余地があります。特に、「女性ユーザーが応募前に知りたい情報を網羅する」視点での拡充が有用です。以下に、具体的な提案をまとめます。

---

## 1. すでにデータモデルにはあるがUIにない項目の補完

### 1-1. transportation_support / housing_support  
送信データには `transportation_support`, `housing_support` フィールドが含まれているようですが、フォーム内ではこれらを直接編集するUIがありません。  
- **提案**: 特徴・待遇(benefits)の下部などに、以下のようなチェックボックスを追加するとわかりやすいです。  
  ```tsx
  <FormField
    control={form.control}
    name="transportation_support"
    render={({ field }) => (
      <FormItem className="flex flex-row items-center space-x-2">
        <FormControl>
          <Checkbox
            checked={!!field.value}
            onCheckedChange={(checked) => field.onChange(checked)}
          />
        </FormControl>
        <FormLabel className="font-normal cursor-pointer">交通費支給あり</FormLabel>
      </FormItem>
    )}
  />
  
  <FormField
    control={form.control}
    name="housing_support"
    render={({ field }) => (
      <FormItem className="flex flex-row items-center space-x-2">
        <FormControl>
          <Checkbox
            checked={!!field.value}
            onCheckedChange={(checked) => field.onChange(checked)}
          />
        </FormControl>
        <FormLabel className="font-normal cursor-pointer">寮・住宅支援あり</FormLabel>
      </FormItem>
    )}
  />
  ```
  これにより、女性ユーザーが「交通費支給」「寮・住宅サポート」の有無をひと目で確認できるようになります。

### 1-2. working_hours / requirements  
`working_hours`, `requirements` というフィールドも送信時に含まれていますが、画面上には入力欄がないように見えます。  
- **提案**: 既にある `application_requirements` とどのように役割を分けるか検討し、もし別用途であればフォーム上にも入力欄を追加しましょう。例えば：
  - `working_hours`: 「1日の勤務可能時間帯」や「シフト制の具体的時間」などを記載する欄  
  - `requirements`: 「応募資格」「年齢制限」「必要書類」など、より制約や必須条件に近い内容  
  - `application_requirements`: より広義の応募要件（歓迎条件やその他条件を含む）

## 2. 女性ユーザー視点で追加すると良い項目

### 2-1. 勤務地・アクセス  
- **最寄り駅や交通手段**  
  - 「◯◯駅から徒歩◯分」「車・バイク通勤OK」「送迎あり」など  
  - Googleマップへのリンク（住所が確定している場合）

### 2-2. 勤務日数やシフト例  
- **週何日からOKか**  
  - 「週1日〜OK」「短期OK」「長期歓迎」など  
- **シフト例の記載**  
  - 「早番：9:00～15:00」「遅番：15:00～22:00」「自由出勤OK」など  
  - 具体的なシフト例があると働くイメージがつかみやすくなります。

### 2-3. 面接・体験入店フロー  
- **応募から面接までの流れ**  
  - 「応募 → 担当者からの連絡 → 面接日程調整 → 面接 → 採用」など  
- **体験入店（体入）の有無**  
  - 体入時の報酬や注意点、必要書類

### 2-4. 制服・衣装・備品について  
- **制服や衣装の有無、貸与の有無**  
  - 「ドレス貸与あり」「私服OK」など  
- **ヘアメイクやネイルOK/NG**  
  - 美容系の自由度を明記しておくと、女性にとって重要な情報です。

### 2-5. セキュリティ・プライバシー対策  
- **個室待機や鍵付きロッカーの有無**  
- **防犯カメラ・セキュリティ体制**  
- **顔出しNG対応**  
  - 「顔写真の掲載は必須ではない」「目線加工OK」など、風俗業界特有のプライバシー配慮

### 2-6. 実際に働くイメージを伝えるコンテンツ  
- **在籍キャストの声・Q&A**  
  - 「未経験でも大丈夫？」「お酒が飲めないけど大丈夫？」など  
- **1日の流れ**  
  - 出勤から退勤まで、どんなスケジュールで働くのか

## 3. UI/UX上の改善ポイント

### 3-1. 入力項目の分割・タブ化  
フォームが長くなる場合、セクションごとにタブやアコーディオンでまとめると見やすくなります。  
- 例: 「基本情報」「給与・待遇」「連絡先情報」「SNS/ウェブサイト」「公開設定」などタブで区切る

### 3-2. 写真ギャラリーの追加  
- **複数枚アップロード**  
  - 店内の雰囲気や待機室など、働く場所をイメージしやすくなる  
  - 既存の `top_image` に加え、`store_gallery` などのフィールドを用意すると良い

### 3-3. 項目の説明ツールチップやヘルプテキスト  
- ユーザーが入力項目の意味や必要性に迷わないように、ツールチップや補足説明を加える

## 4. コード面での補足

1. **手動送信用の関数 (handleManualSubmit) と onSubmit の使い分け**  
   - 両方が存在しており、どちらも最終的に同様の処理をしているため、運用ルールを明確にすると保守性が上がります。  
   - もし `react-hook-form` の標準 `handleSubmit` を使うのであれば、`handleManualSubmit` は不要か、もしくは完全に独立した「バックアップ送信用」などに役割を限定すると良いでしょう。

2. **バリデーションルールの共通化**  
   - `storeProfileSchema` で定義されている Zod スキーマと、手動チェックしている項目（例：電話番号が1つ以上）などが二重管理されている状態です。  
   - Zod スキーマで「電話番号が最低1件は必須」といった条件を盛り込むと、より保守しやすくなります。

3. **各種数値項目のフォーマット**  
   - フロントエンドとバックエンドで整合性を保つために、送信直前に `Number()` で明示的に変換しているのは良い方法です。  
   - ただし、Zod でのバリデーションや UI 表示フォーマット（3桁区切りなど）を行う場合は、そのルールをどこに書くかを明確にしておくと良いでしょう。

---

## まとめ

- **追加すると良い項目**  
  - 「交通費支給」「寮完備」などの有無を示すチェックボックス (transportation_support, housing_support)  
  - 勤務地のアクセス情報、シフト例、面接フロー、制服・ヘアメイクなど女性が気になる詳細情報  
  - セキュリティ面・プライバシー保護の具体策、在籍キャストの声、1日の流れなど実務イメージを伝えるコンテンツ  

- **UI/UX面の工夫**  
  - 長いフォームの場合、タブやアコーディオンで区切って入力しやすく  
  - 画像ギャラリー機能で店舗の雰囲気を伝える  
  - ツールチップや補足説明を加えてユーザーが迷わないようにする  

- **コード・バリデーション面**  
  - 手動送信関数と標準送信関数の使い分けを整理  
  - Zodスキーマで必須条件を一元管理し、二重管理を避ける  

これらを踏まえてフォームを拡張・改善することで、より応募率の高い店舗ページが完成し、女性ユーザーが安心して応募できる環境が整うはずです。
