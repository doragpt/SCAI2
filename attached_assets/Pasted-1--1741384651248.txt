1. 女性向けウェブ履歴書機能
目的
女性ユーザーが自身の詳細なプロフィール（ウェブ履歴書）を簡単に登録・編集し、求人マッチングに活用できるようにすることです。

主な機能
プロフィール入力フォーム
入力項目：
基本情報：氏名、ヨミガナ、生年月日（自動計算で年齢も）、性別など
身体情報：身長、体重、バスト、ウエスト、ヒップ、ブラサイズ（任意入力）
希望条件：勤務期間、前乗り可否、希望保証・単価、待機時間
勤務エリア：出発地、帰宅地、希望地域（NG地域の指定も可能）
経歴・自己PR：過去の勤務経験、自己PR文
その他備考：自由記述欄など
プレビュー機能
入力した内容を最終確認できるプレビュー画面を用意し、誤入力や漏れをチェックできる
下書き保存と編集
ユーザーは途中まで入力した内容を下書き保存し、後から編集可能にする
エラーバリデーション
Zod等のライブラリを用いて、必須項目の入力漏れや形式エラーをリアルタイムに検出
操作フロー
新規登録時

ユーザーは初回ログイン後、マイページに誘導される
「ウェブ履歴書を作成」ボタンをクリック
入力フォーム画面が表示され、必要情報を入力する
各入力欄に対しバリデーションが行われ、エラーがあればその場でフィードバック
入力完了後、プレビュー画面で内容を確認
「保存する」ボタンをクリックし、データはデータベースのtalent_profilesテーブルに保存
既存ユーザーの編集時

マイページ上で「プロフィール編集」ボタンをクリック
登録済みデータがフォームに自動的に反映される
ユーザーは必要に応じて情報を変更
編集後、プレビューで再確認し、最終的に「更新」ボタンで保存
2. 店舗向け求人支援機能
目的
店舗側が低コストで求人情報を掲載し、効率的に適した候補者（女性ユーザー）を検索・採用できるようにすることです。

主な機能
求人情報投稿
店舗は求人投稿フォームから、求人タイトル、仕事内容、勤務時間、給与、応募資格などを入力
月額定額料金（初期は無料期間、後は固定料金または割引料金）で求人掲載が可能
店舗専用ダッシュボード
掲載中の求人情報や応募状況、採用進捗が一覧表示される
登録済みの女性の履歴書（ウェブ履歴書）が一覧で表示され、条件に合致する候補者を検索できる
条件提示と交渉サポート
店舗は候補者に対し、具体的な勤務条件（給与、シフトなど）を提示
AIが条件交渉の補助を行い、双方が合意すれば連絡先が開示される
操作フロー
求人投稿
店舗ユーザーはダッシュボードから「求人を投稿」ボタンをクリック
フォームに求人情報を入力（必須項目と任意項目のチェック）
入力内容を確認後、求人情報がデータベースに保存される
候補者の検索と条件提示
ダッシュボードで登録済みのウェブ履歴書一覧が表示され、フィルター機能で条件検索
候補者を選択し、勤務条件などのオファーを自動もしくは手動で提示
女性側が応募確定またはキャンセルを選択し、双方合意で連絡先が交換される
3. AIマッチング機能
目的
求人条件と女性ユーザーの履歴書情報をもとに、自動で最適なマッチング候補を抽出し、迅速な条件交渉を実現することです。

主な機能
自動マッチングアルゴリズム
初期段階ではルールベース、キーワード照合などの単純な手法でマッチング
将来的には、機械学習モデル（例：BERTなど）を用いて精度向上を図る
マッチングスコアの算出
候補者と求人の各条件（スキル、勤務地、給与希望など）の合致度をスコアリング
一定スコア未満の場合は「マッチなし」として除外する仕組み
キャンセル・評価システム
条件提示後、女性がキャンセルした場合の回数カウント
一定回数キャンセルしたユーザーに対し利用停止や警告を出す仕組み
マッチング成立後、双方が互いに評価を行い、次回以降のマッチングに反映する仕組み
操作フロー
自動マッチングの実行
女性ユーザーがウェブ履歴書を登録・更新すると、その情報が即座にシステムに反映される
店舗側で求人情報が登録されると、システム内部でAIが両者の条件を突合し、マッチング候補を自動抽出
候補者へのオファー送信
マッチング候補として抽出された女性の情報が、店舗のダッシュボードに一覧表示される
店舗は一覧から候補者を選び、条件オファーを送信
応募確定と連絡先開示
女性側がオファー内容を確認し、「応募確定」または「キャンセル」を選択
応募確定の場合、システムは店舗と女性双方に連絡先を開示し、面接や詳細な交渉に進む
4. マッチング導線・応募フロー
目的
ユーザー（女性）と店舗双方が迷うことなく、スムーズにマッチング、応募、条件交渉が行えるようなユーザー体験を提供することです。

主な機能とフロー
ユーザー登録とログイン
女性ユーザーは初回登録時に基本情報とウェブ履歴書を入力
ログイン後、マイページから自分の履歴書を確認・編集できる
店舗ユーザーは自社の求人情報を投稿するため、店舗アカウントでログイン
プロフィール表示と編集
プロフィール表示ページでは、基本情報（ニックネーム、本名、生年月日、希望エリア）とウェブ履歴書（詳細な身体情報や自己PRなど）が表示される
「編集」ボタンから基本情報編集ページに遷移し、再編集が可能
求人情報の検索と応募
女性ユーザーは求人検索機能を利用して、希望条件に合う求人を探す
気に入った求人があれば、応募ボタンをクリックして条件交渉に進む
応募後の流れ
店舗側から提示された条件を女性ユーザーが確認し、応募確定またはキャンセルを選択
応募確定した場合、システムが連絡先を双方に開示し、直接の面談や面接へ移行する
キャンセル・評価システム
条件提示後のキャンセル回数を記録し、一定回数を超えると利用制限などのペナルティを適用
マッチング成立後、双方が評価を行うことで次回以降のマッチングにフィードバックを反映
全体のユーザーフローイメージ
女性ユーザーの流れ

アカウント作成 → ログイン → ウェブ履歴書の作成・編集 → 求人検索 → マッチング候補の表示 → オファー受領・応募確定 → 連絡先開示・面接へ → 評価入力
店舗ユーザーの流れ

店舗アカウント作成 → ログイン → 求人情報の投稿 → 応募者リストの閲覧 → 候補者への条件提示 → 応募確定後、連絡先の開示 → 面接・採用の実施 → 評価入力
これらのフローと機能は、システム全体として以下の点で統合されています。

データの一元管理
女性ユーザーの入力情報はtalent_profilesテーブルに、店舗の求人情報は別テーブルに保存され、双方がAPI経由で取得されます。
自動化と効率化
AIによるマッチングと自動交渉補助により、従来の手作業での連絡や調整を大幅に削減。
安全性と信頼性
認証・認可、エラーハンドリング、キャンセル制限、評価システムなど、トラブル防止対策も盛り込まれています。
